🏗️ Architecture Principles

Master Data Integrity

All masters enforce referential integrity. Cannot delete a master record if child records exist in other tables. Soft deletes via active/inactive flags for audit trails.

Audit Trail

Every master table includes created_at, created_by, updated_at, updated_by timestamps. Changes to critical fields trigger audit log entries.

Hierarchical Organization

Masters organized into 6 logical categories. Each category groups related data for easier navigation and permission management.

Extensibility

Custom Fields Registry allows unlimited user-defined fields on any master without schema changes. Configuration-driven approach.

Multi-Tenancy Ready

All masters include tenant_id for multi-organization deployments. Master data scoped by organization by default.

Performance

Frequently referenced masters (Customer, Supplier, Product) are cached at application layer. Indexed on primary lookups and status fields.
